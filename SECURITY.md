# 보안 및 쿠키 안내 (반드시 읽어주세요)

이 앱은 리디북스 공개 API가 없어서 **당신의 로그인 쿠키(`ridi-at`)** 로 리디 API를 대신 호출합니다.
이 쿠키는 사실상 **당신 계정의 열쇠**입니다. 아래 내용을 이해하고 사용하세요.

---

## 1. `ridi-at` 쿠키가 뭐고, 뭘 할 수 있나 (⚠️ 책임 강조)

`ridi-at`은 리디 계정의 **액세스 토큰(JWT)** 입니다. `scope: all` 로 발급되며 유효기간은 약 **24시간**입니다.
이 값을 가진 사람은 만료 전까지 **당신인 것처럼** 다음을 할 수 있습니다.

- 서재 전체 · 구매 내역 · 읽기 기록 · 결제/주문 관련 정보 열람
- 계정 개인정보 접근, 상황에 따라 **구매·주문 등 계정 조작**
- 즉, **당신 계정 도용**

> **그래서 이 토큰의 관리 책임은 100% 당신에게 있습니다.**
> - 신뢰할 수 없는 사이트·앱·확장프로그램에 붙여넣지 마세요.
> - 코드/깃/스크린샷/채팅에 노출하지 마세요. (`data/`, `.env`는 이미 git에서 제외됨)
> - **아무나 접근 가능한 서버에 배포하지 마세요.** 배포하는 순간 그 서버를 통해 토큰이 유출될 수 있습니다.
> - 유출이 의심되면 즉시 아래 [4. 쿠키 초기화](#4-쿠키-초기화무효화)를 실행하세요.

`cf_clearance`는 Cloudflare 통과용 토큰(당신의 IP/브라우저에 묶임)으로, 차단이 있을 때만 보조로 씁니다. 민감도는 `ridi-at`보다 낮지만 함께 관리하세요.

---

## 2. 이 앱은 쿠키를 어떻게 처리하나

- 입력한 `ridi-at`은 **이 앱의 서버(당신 컴퓨터에서 도는 Next.js)** 로만 전송됩니다.
- 서버에서 **AES-256-GCM으로 암호화**해 `data/sessions/<세션id>.json`에 저장합니다.
  암호화 키는 `RD_SECRET` 환경변수 또는 최초 실행 시 생성되는 `data/.key`입니다.
- 리디 API 호출 시에만 서버 메모리에서 복호화해 사용하고, **브라우저(클라이언트)로 다시 내려보내지 않습니다.**
- 브라우저에는 토큰 대신 **httpOnly 세션 쿠키(`rd_sid`)** 만 남습니다. (JS로 못 읽음)
- 토큰을 로그로 남기지 않습니다.

한계: 같은 컴퓨터에 접근할 수 있는 사람은 `data/` 폴더(암호화 blob + `.key`)를 함께 가져가면 복호화할 수 있습니다. **로컬 실행 + 본인만 쓰는 기기**가 전제입니다.

---

## 3. 쿠키 추출 방법

### 방법 A — 개발자도구 (확장프로그램 없이, 가장 안전 · 권장)
1. 브라우저에서 `ridibooks.com` 로그인
2. `F12` → **Application(애플리케이션)** 탭 → 좌측 **Cookies** → `https://ridibooks.com`
3. `ridi-at` 값을 복사해 앱에 붙여넣기 (차단되면 `cf_clearance`도)

### 방법 B — 쿠키 추출 확장프로그램
확장프로그램은 편하지만 **그 자체가 쿠키를 훔쳐갈 수 있는 권한**을 가집니다. 평판 좋은 것만, 필요 후엔 제거하세요.
- **Cookie-Editor** (Chrome/Edge/Firefox) — 가장 널리 쓰이고 관리가 활발함. 권장.
- **EditThisCookie** — 유명하지만 관리가 뜸하고 사칭 클론이 많음 → 설치 시 개발자/리뷰 확인 필수.
- 어떤 확장이든 **오프라인·오픈소스·리뷰 많은 것**을 고르고, 추출이 끝나면 **비활성화/삭제**하는 걸 권장합니다.

> 잘 모르겠으면 **방법 A(개발자도구)** 를 쓰세요. 제3자 코드가 개입하지 않습니다.

## 4. 쿠키 초기화/무효화

유출 의심 시 또는 사용 종료 시:

1. **앱에서 로그아웃** — 우측 상단 `로그아웃`. 서버의 세션 파일(`data/sessions/*`)이 삭제되고 `rd_sid` 쿠키가 제거됩니다.
2. **로컬 데이터 완전 삭제** — 앱 폴더에서:
   ```bash
   rm -rf data     # 암호화된 토큰 + 스냅샷 + 암호화키(.key) 모두 삭제
   ```
3. **리디 쪽 토큰 무효화** (가장 확실) — 아래 중 하나:
   - 브라우저에서 **ridibooks.com 로그아웃** → `ridi-at`/`ridi-rt`가 무효화됨
   - **리디 비밀번호 변경** → 발급된 모든 세션/토큰 무효화
   - 리디 **계정 설정의 기기/로그인 관리**에서 기기 로그아웃
4. 그냥 두어도 `ridi-at`은 **약 24시간 뒤 자동 만료**됩니다. (하지만 유출 시엔 1·2·3을 즉시 하세요.)

## 5. 배포에 대해 (특히 Vercel)

**기본 권장: 로컬 실행(`npm run dev`).** 토큰이 당신 기기 밖으로 안 나갑니다.

배포하려면 위험과 플랫폼 특성을 이해해야 합니다.

> **실측 확인:** `https://ridi-driller.vercel.app` 배포는 로그인 시 **502**가 납니다.
> 같은 `ridi-at` 토큰이 **로컬(가정용 IP)에선 200으로 성공**하지만 Vercel(데이터센터 IP, 예: `sin1` 싱가포르)에선 실패합니다.
> 원인은 **리디의 Cloudflare가 데이터센터/클라우드 IP를 차단/챌린지**하기 때문입니다. 코드·토큰 문제가 아니라 **호출 IP 문제**라 Vercel에선 근본적으로 동작하지 않습니다.
> (설령 이걸 우회해도 아래의 파일시스템·타임아웃 문제가 남습니다.)

### Vercel 특징 (이 앱과 잘 안 맞는 점)

- **Cloudflare가 데이터센터 IP 차단** (가장 치명적): 위 실측대로, Vercel 서버에서 리디 API 호출이 Cloudflare에 막힙니다. KV/DB로 바꿔도 이건 해결 안 됩니다.
- **서버리스·임시 파일시스템**: 함수 실행마다 파일시스템이 초기화/읽기전용에 가깝습니다. 이 앱은 `data/`에 세션·스냅샷을 **파일로 저장**하므로 Vercel에선 **저장이 유지되지 않아 로그인/캐시가 깨집니다.** (Vercel KV/Postgres 같은 외부 저장소로 바꿔야 함)
- **실행 시간 제한**: Hobby 플랜은 함수 최대 실행이 짧습니다(대략 10~60초). 이 앱의 **전체 동기화는 2~3분**이라 **타임아웃**납니다. (백그라운드 잡/큐가 필요)
- **공개 URL**: 기본이 공개입니다. 온보딩 페이지가 노출되고, 무엇보다 **배포된 인스턴스에 당신의 암호화 토큰이 올라갑니다.** 그 서버/환경변수(`RD_SECRET`)에 접근 가능한 사람은 토큰을 복호화할 수 있습니다.

### 그래도 배포한다면
- **오직 당신만 접근 가능**하게 하세요 (접근 인증 추가, 비공개 네트워크 등).
- 파일 대신 **영구 저장소(DB/KV)** 로 교체하고, 긴 동기화는 **백그라운드 처리**로 바꾸세요.
- `RD_SECRET`을 강력한 값으로 설정하고 절대 커밋하지 마세요.
- 배포는 곧 "내 계정 열쇠를 남의 인프라에 보관"하는 일임을 기억하세요. **위험과 책임은 당신의 몫입니다.**

## 요약

- `ridi-at` = 내 리디 계정 열쇠(약 24h). 관리 책임은 전적으로 **나**.
- 이 앱은 로컬 서버에 **암호화 저장**, 클라이언트로 재노출 안 함. 하지만 로컬/기기 보안이 전제.
- 추출은 **개발자도구가 가장 안전**. 확장은 Cookie-Editor 정도, 쓰고 나서 제거.
- 유출·종료 시: 앱 로그아웃 → `rm -rf data` → 리디 로그아웃/비번 변경.
- **Vercel 등 공개 배포는 비권장.** 하려면 저장소·타임아웃·접근제어를 반드시 손봐야 하고, 유출 위험을 감수해야 함.
